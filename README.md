# Academic Projects

## Overview
このリポジトリは、大学の情報工学系のカリキュラムにおいて、フレームワークやテンプレートに依存せず、ゼロベースで設計・実装を行ったプロジェクトをまとめたものである。
アルゴリズムの理解、低レイヤーアーキテクチャの操作、および数学的解析に基づく実装力の証明として公開している。

## Projects

各ディレクトリの詳細な仕様や実行方法については、それぞれのディレクトリ内のREADME.mdを参照。

### 1. calc-compiler
* **概要:** C言語による構文解析から、実際の計算を実行するx86-64アセンブリコードの生成までをフルスクラッチで行う電卓コンパイラ。
* **技術スタック:** C, x86-64 Assembly, GCC
* **ハイライト:** `imul` / `idiv` 命令を一切使用せず、2の補数の性質やビット操作（シフト演算と加減算）のみで乗除算を再構築した（calc3）。さらに再帰関数とスタック（`pushq`/`popq`）を駆使し、レジスタの上書きを防ぎながらカッコ `()` を用いた優先計算を実装している。

### 2. 2d-game
* **概要:** ターミナル上で動作する、タイルベースの2Dダンジョン自動生成および探索シミュレータ。
* **技術スタック:** C 
* **ハイライト:** 部屋の座標ソートとL字接続アルゴリズムにより、完全に繋がった「詰みのない」ダンジョンを動的に生成する。また、標準の `malloc` / `free` による断片化を防ぐため、独自のメモリプールを実装してエンティティの生成と論理削除を高速化。バイナリデータでの状態保存・復元機能も備えている。

### 3. Image-Rotation-Mathematical-Analysis
* **概要:** OpenCV等の回転関数に依存せず、NumPyを用いた行列演算により画像の回転処理と補間アルゴリズムを数学的に実装・検証したプロジェクト。
* **技術スタック:** Python, NumPy, Jupyter Notebook
* **ハイライト:** アフィン変換の順マッピングによる「穴あき問題」を逆マッピングで解決し、さらに最近傍補間とバイリニア補間を自作。OpenCVの最適化関数と実行速度およびMSE（平均二乗誤差）の比較検証を行い、アルゴリズムによる画質と計算コストのトレードオフを考察している。

## Environment & Run
各プロジェクトは独立しており、C言語プロジェクトはGCCコンパイル、PythonプロジェクトはJupyter環境での実行を想定している。
