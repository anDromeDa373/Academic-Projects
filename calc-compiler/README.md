# 電卓コンパイラ

C言語を用いて構文解析を行い、実際の計算を実行するx86-64アセンブリコードを生成する電卓コンパイラの実装である。入力の解釈と計算処理を分離することで、演算ロジックの実装に集中した設計となっている。

---

## 各バージョンの機能と実装

本プログラムは段階的に機能を拡張して実装されている。

### calc1 (基本機能の実装)
* 四則演算およびメモリ機能（M+, M-, MR, MC）を実装。
* CPUの乗算命令（`imul`）および除算命令（`idiv`）を使用。
* 演算処理の優先順位は考慮せず、入力された順に左から計算を実行する。オーバーフロー検知は行わない。
* 符号キー（`S`）による入力数値の正負反転に対応。

### calc2 (堅牢性の向上)
* `calc1`の機能に加え、演算命令直後の条件分岐命令（`jo`）を用いたオーバーフロー検知とエラー処理（`E`の表示および実行終了）を追加。
* プロセスのクラッシュを防ぐため、ゼロ除算および `-2^31 / -1` のオーバーフローケースを演算前に事前検査する。
* 32ビット符号あり整数の最小値（`-2^31`）の入力に対応するため、数値を負の数として累積し、最後に符号の整合性を合わせるアルゴリズムを採用。

### calc3 (乗除算命令を用いない実装)
* `imul` および `idiv` 命令を一切使用せず、ビット操作と加減算によって乗除算を完全に再構築。
* 数値入力時の「10倍」の処理をビットシフトを用いた分解（`N * 8 + N * 2`）に置き換え、低コスト化を実現。
* 負の数の乗算において、2の補数の性質とオーバーフロー時の上位ビット切り捨てを活用し、冗長な条件分岐や正負判定ロジックを排除。除算においては、スタックを用いた符号フラグの保存と復元により対応した。

### calc4 (拡張機能: カッコへの対応)
* `calc1`の機能をベースに、カッコ `()` を用いた優先計算を追加実装。
* 再帰関数を用いて、カッコ内の式を独立した一つの数式として解釈する。
* カッコ内の処理へ移行する際、それまでの計算結果（`%eax`）をスタック（`pushq`）に退避し、計算終了後に復元（`popq`）することで、レジスタの上書きを防止しながら階層的な計算を実現した。

## 動作・開発環境

* **アーキテクチャ:** x86-64
* **コンパイラ:** GCC (Cコードのコンパイル、および生成されたアセンブリコードの実行)
* **シェル:** csh (自動テストスクリプトの実行)
*(※x86-64アーキテクチャのLinux/Unix系OSでの実行を想定)*

---

## レジスタ設計

| レジスタ | 用途 |
| :--- | :--- |
| **`%eax`** | 演算の累積結果を常に保持する |
| **`%ebx`** | 演算対象となる数値を一時的に保持する |
| **`%ecx`** | 文字列解析時の数値構築、乗除算の商の格納、カッコ内計算結果の移動に使用 |
| **`%edx`** | 入力文字の保持、および除算時の余り計算用変数として使用 |

## 制限と今後の課題

* C言語コード内にアセンブリ命令を出力する `printf` が直接多数記述されており、構文解析ロジックとコード出力の分離が不十分であるため、拡張性に課題がある。
* 演算子の直後に入力された符号キーを無視する仕様のため、数式の中に直接負の数を記述する入力（例: `12*-2` を `12*S2` のように入力する場合）が直感的に動作しない。
* `calc3`の自作乗算処理において、オーバーフロー時の切り捨て挙動に依存した実装となっている。他の環境での整合性や、負の数（例: `-1` を `2^32 - 1` として処理する）を扱う際のループ回数が最大32回に達するなど、計算コスト面での非効率性が残っている。
