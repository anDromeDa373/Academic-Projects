# Random 2D Game World Simulator

## 概要
本プロジェクトは、タイルベースの2Dゲーム世界をランダムに自動生成し、ターミナル上で探索やアイテム収集を行えるシミュレータである。C言語を用いて実装しており、独自のメモリプールを活用した高速なエンティティ管理と、バイナリ形式による堅牢なセーブ・ロード機能を特徴とする。

## 特徴と実装の工夫

* **ダンジョン自動生成アルゴリズム**
  完全なランダム配置による「詰み」を防止するため、部屋を基準とした生成アルゴリズムを実装した。初期位置を含む部屋を作成後、重ならないように複数の部屋を配置し、それらの中心x座標をソートしてL字型の通路で接続することで、すべての部屋が確実に繋がる探索可能なダンジョンを生成する。
* **メモリプールによるエンティティ管理**
  標準の `malloc` / `free` によるメモリの断片化や速度低下を防ぐため、`db_engine`モジュールにて固定サイズのメモリブロックを事前確保し、管理する方式を採用した。これによりエンティティの高速な生成と、フラグを用いた安全な論理削除を実現している。
* **セーブ＆ロード機能**
  実行中のゲーム状態（マップサイズ、プレイヤーの座標・HP、インベントリ、各エンティティのステータス）をコンパクトなバイナリデータとしてファイルに保存し、完全に復元する機能を備える。

## 遊び方とコマンド
プレイヤーはマップの左上にスポーンする。以下のコマンドを入力してゲームを進行する。移動コマンドは複数文字をまとめて入力することで連続移動が可能である（例: `dddw`）。

### コマンド一覧
* `w` / `a` / `s` / `d` : プレイヤーの移動（上 / 左 / 下 / 右）
* `l` : 周囲のマスの状況を調べる（隣接するタイルの情報を表示）
* `i` : インベントリの確認（所持アイテムと現在HPを表示）
* `u` : 回復ポーションを使用してHPを全回復
* `save <filename>` : 現在のゲーム状態をファイルに保存
* `load <filename>` : 保存したゲーム状態をファイルから復元
* `quit` : ゲームを終了

### マップの記号
* `P` : プレイヤー（初期HP: 3）
* `E` : 敵（衝突するとプレイヤーのHPが1減少し、敵は消滅する）
* `I` : アイテム（回復ポーション。取得後自動でインベントリに入り、`u`コマンドで使用可能）
* `#` : 壁
* `.` : 空きマス（床）

## 動作環境とビルド方法
本プログラムは追加の外部ライブラリを使用していない。以下の手順でコンパイルおよび実行が可能である。

### 通常ビルドと実行
```bash
gcc -std=c11 -Wall -Wextra -I. db_engine.c entity_manager.c dungeon_gen.c world.c main.c -o gameworld
./gameworld new_world.dat
