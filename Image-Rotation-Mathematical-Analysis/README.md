# Image Rotation and Interpolation from Scratch

## 概要
本リポジトリは、画像処理における「画像の回転処理」の数理的背景を理解し、外部ライブラリ（OpenCVなど）の回転関数に依存せず、NumPyを用いてゼロから実装した学習プロジェクトです。（大学2年次 Q1課題）

Jupyter Notebook (`.ipynb`) 上で、回転行列を用いた座標変換や逆マッピングの理論を実装し、異なる補間アルゴリズム（最近傍補間・バイリニア補間）が画質や処理速度に与える影響を検証しています。

## ファイル構成
- `main.ipynb`: 回転処理の実装、実行時間・精度の検証、考察をまとめたノートブック
- `honkan.jpg`: テスト用の入力画像

## 実装したアルゴリズム
以下の3つのアプローチで画像の回転処理を実装し、OpenCVの `warpAffine` と比較を行いました。

1. **順マッピング ＋ 最近傍補間**
   - 元画像の座標を回転行列で変換する基礎的な手法。変換先の非整数座標の丸め処理により、画像に「穴」が生じる問題を確認。
2. **逆マッピング ＋ 最近傍補間**
   - 出力画像の座標から元画像の座標を逆算し、順マッピングの穴あき問題を解決した手法。
3. **逆マッピング ＋ バイリニア補間**
   - 逆マッピング時に周囲4ピクセルの値を用いた重み付き平均を計算し、より滑らかな回転画像を出力する手法。

## 実行結果と考察のサマリー
ノートブック内での検証により、以下の結果を得ました。

* **画質と精度**: 順マッピングでは無数の穴が発生しますが、逆マッピングにより解決されます。バイリニア補間を適用することで、視覚的にも滑らかになり、MSE（平均二乗誤差）の観点でも最も精度が高くなる傾向を確認しました。
* **処理速度**: 現在の手作りの実装では、画像サイズに対するループ処理の影響が大きく、C++ベースで高度に最適化されたOpenCVと比較すると実行時間に大きな差が生じました。

## 今後の技術的な課題
本実装はアルゴリズムの基礎理解を優先したため、実用的なエンジニアリングの観点では以下の課題が残っています。

1. **ベクトル化による高速化**: 現在の実装はPythonのネイティブな `for` ループに依存しているため、`numpy.meshgrid` 等を活用した行列表算（ベクトル化）にリファクタリングし、処理速度を向上させる。
2. **厳密な精度評価**: 手作りの関数とOpenCVの関数で「回転の中心座標」やオフセットの扱いを完全に統一し、サブピクセル単位でのズレをなくした上で、より厳密なMSE比較を行う。
